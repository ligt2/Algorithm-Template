## 一、问题识别：题目特征 → 图论模型

### 1.1 关键词 → 模型速查表

```
┌───────────────────────────────────────────────────────────────────────┐
│  题目特征/关键词                          图论模型          用什么模板  │
├───────────────────────────────────────────────────────────────────────┤
│  "最短路"/"最少花费"/"最小时间"           最短路            Dijkstra   │
│  "连通"/"合并"/"分组"/"朋友圈"           并查集            DSU        │
│  "依赖关系"/"先后顺序"/"课程安排"         DAG+拓扑排序      TopoSort   │
│  "环"/"强连通"/"互相可达"                 SCC              Tarjan     │
│  "匹配"/"配对"/"二选一分组"               二分图            Bipartite  │
│  "最大流量"/"网络容量"                    网络流            Dinic      │
│  "树上距离"/"公共祖先"                    LCA              LCA        │
│  "n个点n条边"/"每个点一条出边"            基环树            TreeDiff   │
│  "割开"/"桥"/"关键点"                     割点割边          Tarjan     │
│  "一笔画"/"不重复走遍"                    欧拉路径          Euler      │
├───────────────────────────────────────────────────────────────────────┤
│  【隐式图】需要建模才能发现                                           │
├───────────────────────────────────────────────────────────────────────┤
│  "A说B是坏人"/"A和B矛盾"                  二分图/并查集     染色/扩展域 │
│  "x选了则y必选"/"蕴含关系"                2-SAT            Tarjan     │
│  "状态转移"/"最少步数"                    BFS最短路         -          │
│  "集合包含/依赖"                          DAG              TopoSort   │
│  "真假判断"/"说谎者问题"                  基环树/染色       具体分析    │
└───────────────────────────────────────────────────────────────────────┘
```

### 1.2 隐式建图的三个信号

```
┌─────────────────────────────────────────────────────────────────┐
│  信号1：存在"关系"描述                                           │
│  例："A认为B是..."、"A依赖B"、"A和B相邻/相连/冲突"              │
│  → 关系就是边！A和B之间连一条边                                  │
├─────────────────────────────────────────────────────────────────┤
│  信号2：存在"状态转移"                                           │
│  例："从状态A可以变成状态B"、"操作后..."                        │
│  → 状态是点，转移是边，求最短路                                  │
├─────────────────────────────────────────────────────────────────┤
│  信号3：存在"约束/蕴含"                                          │
│  例："如果x那么y"、"x和y不能同时选"                             │
│  → 2-SAT/二分图模型                                              │
└─────────────────────────────────────────────────────────────────┘
```

## 二、真题实战：Nailoong问题详解

### 2.1 题目翻译

```
n个生物，第i个说"第a[i]个是Nailoong"
规则：
- Nailoong总是说谎
- 其他生物可能说真话或假话
求：最多有多少个Nailoong
```

### 2.2 逻辑推导 → 图模型

```
【关键推导】
如果 i 是 Nailoong → i说谎 → a[i] 不是 Nailoong
如果 i 不是 Nailoong → i可能说真或假 → 对a[i]无约束

【转换为图】
建边：i → a[i]（每个点恰好一条出边）
约束：如果选i为Nailoong，则不能选a[i]

【识别模型】
- 每个点恰好一条出边 → 这是【内向基环树】！
- 约束"选i则不选a[i]" → 这是【最大独立集】问题！
```

### 2.3 结构分析

```
内向基环树的结构：
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│      树1         树2                                        │
│       ↓           ↓                                         │
│   ┌───────────────────────┐                                │
│   │    环（核心部分）      │                                │
│   │   A → B → C → D → A   │                                │
│   └───────────────────────┘                                │
│       ↑                                                     │
│      树3                                                    │
│                                                             │
│  【处理方法】                                                │
│  1. 找出所有环                                               │
│  2. 对挂在环上的树：树形DP求最大独立集                        │
│  3. 对环本身：环形DP（断环为链，分两种情况）                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.4 算法设计

```cpp
/*
  【用到的DP】基环树DP = 树形DP + 环形DP
  
  1. 拓扑排序找环（度为1的点不断删除，剩下的是环）
  2. 对每棵挂在环上的子树：
     dp[u][0] = 不选u时，u子树的最大独立集
     dp[u][1] = 选u时，u子树的最大独立集
  3. 对环上的点：
     - 枚举环上第一个点选或不选
     - 进行链式DP
     - 取两种情况的最大值
*/
```

### 2.5 样例验证

```
输入：n=6, a=[4,1,1,5,6,5]

建图（i→a[i]）：
1→4, 2→1, 3→1, 4→5, 5→6, 6→5

结构分析：
- 环：5→6→5（长度2的环）
- 树：1←2, 1←3, 4←1, 5←4

环上：5和6不能同时选，最多选1个
树上：在满足约束下最大化

答案：4
```

## 三、图论问题的性质推导框架

### 3.1 常用图性质

```
┌─────────────────────────────────────────────────────────────────┐
│  性质                         应用场景                          │
├─────────────────────────────────────────────────────────────────┤
│  树有n-1条边                  判断是否是树/森林                  │
│  n点n边 → 恰好1个环           基环树识别                        │
│  二分图 ↔ 无奇环              染色判定、二分图匹配               │
│  DAG ↔ 无环                   拓扑排序、DP                      │
│  最大流=最小割                 网络流对偶                        │
│  二分图最大匹配=最小点覆盖     König定理                         │
│  最大独立集=n-最大匹配         二分图独立集                      │
│  SCC缩点后是DAG               有向图处理                        │
│  欧拉回路 ↔ 所有点度数为偶    一笔画判定                        │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 推导思路模板

```
【从约束到图】
Step 1: 找出题目中的"对象"和"关系"
        对象 → 点
        关系 → 边

Step 2: 分析关系的性质
        - 单向还是双向？ → 有向/无向
        - 有权重吗？ → 带权图
        - 关系是什么含义？ → 决定边的语义

Step 3: 转化为已知问题
        - 求最优解 → 最短路/最大流/DP
        - 求可行性 → 连通性/2-SAT
        - 求划分 → 二分图/染色

Step 4: 检查特殊结构
        - 每个点出度=1？ → 基环树
        - 能二染色？ → 二分图
        - 无环？ → DAG
```

## 四、常见图论建模套路

### 4.1 说谎者/真假问题 → 图染色或基环树

```
【套路】
- "A说B是坏人" → 建边 A→B
- 说谎者说的话取反，老实人说的话可能对可能错
- 通常转化为：选最多的说谎者，使得约束不矛盾

【例题】本题Nailoong
```

### 4.2 "选A则必选B"/"A和B冲突" → 2-SAT

```
【套路】
- x选了则y必须选 → ¬x ∨ y → 建边 x→y, ¬y→¬x
- x和y冲突 → 建边 x→¬y, y→¬x
- 用Tarjan求SCC判断是否有解
```

### 4.3 "最少操作次数"/"状态转移" → BFS最短路

```
【套路】
- 每个状态是一个点
- 每次操作是一条边（权为1）
- 求最少操作 = 求最短路（BFS即可）
```

### 4.4 "分成两组"/"二选一" → 二分图

```
【套路】
- A和B不能同组 → 连边A-B
- 检查是否能二染色
- 每种颜色是一组
```

### 4.5 "依赖顺序"/"必须先做A再做B" → DAG+拓扑

```
【套路】
- A必须在B之前 → 建边 A→B
- 拓扑排序得到合法顺序
- DAG上DP求最长路/方案数
```

## 六、专科组图论重点

### 6.1 高频考点（必须熟练）

```
⭐⭐⭐⭐⭐ 必考
├─ 最短路（Dijkstra为主）
├─ 并查集（连通性判断）
├─ 拓扑排序（依赖关系）
└─ 基础建图能力

⭐⭐⭐⭐ 常考
├─ 最小生成树
├─ 二分图判定和匹配
├─ BFS求最短路（状态图）
└─ 简单的树上问题

⭐⭐⭐ 可能考
├─ SCC/缩点
├─ LCA
├─ 基环树
└─ 最大流（简单建模）
```

### 6.2 DP与图论的结合

```
┌─────────────────────────────────────────────────────────────────┐
│  图结构 + DP类型                                                 │
├─────────────────────────────────────────────────────────────────┤
│  DAG + 线性DP     →  DAG最长路/最短路/方案数                     │
│  树 + 树形DP      →  树的直径/最大独立集/背包                    │
│  基环树 + 环形DP  →  断环为链，分类讨论                          │
│  状态图 + BFS     →  最少操作步数                                │
│  二分图 + 匹配    →  最大匹配/最小覆盖                           │
└─────────────────────────────────────────────────────────────────┘

【本题用到的DP】
基环树DP = 树形DP（最大独立集）+ 环形DP（断环分类）
```

## 七、快速识别练习

### 给你几道题，练习识别：

```
题1："n个城市，m条路，每条路有长度，求1到n最短距离"
     → 【最短路】Dijkstra

题2："n个人，如果a和b是朋友，b和c是朋友，则a和c也是朋友，求朋友圈数量"
     → 【连通分量】DSU

题3："n门课，有些课必须先修，求一个合法的学习顺序"
     → 【拓扑排序】TopoSort

题4："n个人分成两队，有些人不能同队，问能否分成功"
     → 【二分图染色】BipartiteCheck

题5："每个人指向另一个人，求最多能选多少人，选的人不能指向被选的人"
     → 【基环树+最大独立集】TreeDiff + 树形DP（就是本题！）

题6："从状态A变成状态B，每次可以操作x/y/z，最少几步"
     → 【状态图BFS】最短路
```

## 八、总结：三步走

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  【第一步】识别                                                  │
│  读题 → 找"对象"和"关系" → 识别图类型 → 匹配模板                 │
│                                                                 │
│  【第二步】建图                                                  │
│  确定点是什么 → 确定边是什么 → 确定边的方向和权值                 │
│                                                                 │
│  【第三步】求解                                                  │
│  套模板 → 修改细节 → 输出答案                                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

核心能力：从题目描述 → 抽象出图模型 → 识别是哪类经典问题
这个能力只能通过多做题来提升，但有了这个框架会快很多！
```
